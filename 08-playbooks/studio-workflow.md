# Studio Workflow

## Workflow commun
1. Brief entrant
2. Clarification rapide
3. Cadrage Product
4. Architecture / faisabilité CTO
5. Production (SaaS / site / landing / asset)
6. QA / revue interne
7. Preview ou livrable
8. Feedback Titouan
9. Itération
10. Validation finale

## Variante SaaS
- brief feature
- spec
- architecture
- build
- tests
- preview
- review

## Variante landing page
- brief offre/angle/style
- structure de page
- copy + design
- preview
- review
- déploiement

## Variante site client
- intake client
- brief marque/service/zone
- structure + contenu + assets
- preview
- validation
- déploiement domaine

## Règle de fluidité
Titouan ne doit intervenir que pour :
- clarifier si nécessaire
- valider les points importants
- reviewer les sorties
