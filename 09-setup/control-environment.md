# Environnement recommandé pour piloter tous les agents

## Ce qu'il te faut
1. Telegram pour le quotidien rapide
2. Dashboard / Control UI OpenClaw pour la vue d'ensemble locale
3. Le workspace Git comme source de vérité
4. Plus tard : un repo produit + éventuel board projet (GitHub Issues/Projects)

## Vue d'ensemble recommandée
### A. OpenClaw Dashboard
- voir les sessions
- voir l'activité
- voir le workspace plus facilement que dans Telegram
- utile pour piloter localement

### B. Workspace structuré
- tous les fichiers agent/projet ici
- chaque création importante doit laisser une trace fichier

### C. Registry + logs projet
- `10-control-tower/agents-registry.md`
- `10-control-tower/tasks/`
- `10-control-tower/handoffs/`
- `10-control-tower/decisions.md`

## Principe
Telegram = interface de conversation
Workspace + Dashboard = interface de pilotage
