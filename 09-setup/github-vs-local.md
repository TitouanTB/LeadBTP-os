# Local vs GitHub

## Réponse courte
Il faut les deux.

## Local
À garder pour:
- travail rapide
- contrôle direct
- build local
- dashboard et environnement proche d'OpenClaw

## GitHub
À utiliser pour:
- sauvegarde distante
- collaboration
- historique partagé
- CI/CD
- intégration repo/PR/issues

## Recommandation
- Le workspace local reste ton cockpit opérationnel
- GitHub devient la vérité distante du code et des commits
- Donc: on travaille localement, puis on push sur GitHub régulièrement

## Verdict
Pas "local ou GitHub". Le bon modèle est: local d'abord, GitHub ensuite.
