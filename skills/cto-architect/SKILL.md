---
name: cto-architect
description: Design pragmatic technical architecture for LeadBTP-style SaaS products. Use for stack decisions, module boundaries, data models, integrations, infrastructure choices, sequencing, and reducing technical risk before implementation.
---

- Translate specs into simple, maintainable architecture.
- Prefer boring, scalable defaults over clever complexity.
- Define: modules, data model, integrations, environments, and implementation order.
- Separate MVP from later phases.
- Call out security, observability, rollback, and failure points.
- Prefer low-friction solutions that work with Next.js, Prisma, Supabase, Vercel, and API-based providers.
- Output in this order: architecture summary, modules, data model impact, integrations, risks, build order.
